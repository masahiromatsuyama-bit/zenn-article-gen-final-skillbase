## はじめに

Anthropicが16のClaude Codeエージェントを2週間並列実行し、人間の介入なしで10万行規模のRust製Cコンパイラを構築したという事例から、AIが長時間自律作業を続けられる秘訣は「Agent Loopの理解とフィードバックループの設計」にあることを解説する記事です。

## フィードバックループとは何か

フィードバックループは「行動した結果を評価し、その結果をもとに次の行動を修正する循環構造」を指します。エアコンが室温を測定→設定温度と比較→出力調整する仕組みと同じく、ソフトウェア開発でも「コード作成→テスト実行→失敗時修正」というサイクルが品質向上を実現します。

アジャイル開発はこの概念の実践例で、スプリント計画→実装→レビュー→レトロスペクティブ→改善というループが不確実性の高いプロジェクトでも前進を可能にします。

## Agent Loopとは何か

Agent Loopは、AIエージェントが「推論・ツール利用・結果観測」のサイクルを自律的に繰り返し、目標達成まで思考-行動-観察を反復・修正し続けるプロセスです。これはReAct（Reasoning + Acting）とも呼ばれます。

単なるプロンプト応答との違いは、一回の「思考→応答」ではなく、思考→行動→評価→再思考を繰り返すループ構造にあります。

## Agent Loopの構造

基本プロセスは：

1. ユーザー入力受け取り
2. 思考（推論）
3. ツール選択と実行
4. 結果反映と再思考
5. 終了判定

### パフォーマンス最適化

**プロンプトキャッシュ:** ループが継続するにつれプロンプトが大きくなるため、同じ内容は再計算を省く仕組みが導入されています。

**コンテキスト圧縮:** 長いループでは会話履歴を要約し、不要な部分を縮小してコンテキストを節約します。

## Agent Loop設計とは何か

Agent Loop設計は、AIが自律的に行動→評価→改善しながら目標達成に向かう自己循環型の仕組みを、人間側でフィードバックループを設計し環境を整えることです。OpenAIではこれを「ハーネスエンジニアリング」と呼びます。

### 「目標を理解する」ことの重要性

「Cコンパイラを作る」という目標は曖昧です。以下が定義されていなければなりません：

- どのC規格に準拠するか
- どのテストスイートを全通させるか
- 許容できる警告レベル
- 最適化レベル
- 対象プラットフォーム

AIは「雰囲気」でゴールを判断できません。具体的な数値目標を設定すると、AIはボトルネック調査→計測コード追加→仮説立案→効果測定を繰り返すようになります。

### テストで自己修正ループを回す

長時間自律開発において、最強のフィードバック装置はテストです。テストがあると、AIは自動的に「実装→テスト実行→失敗検出→原因分析→修正→再テスト→収束」という自律的フィードバック制御を実現します。

## 内部ループ vs メタループ

**内部ループ:** 1回のClaude Code実行内での思考と修正サイクル

**メタループ:** セッションを超えた長期自己改善

長時間作業では、文脈揺らぎ、調査計画スキップによる破綻、評価軸欠落といった問題が発生します。

## Human in the Loopを排除する

長時間自律実行には「人間が毎回介入しなくても回る構造」が必須です。

**重要なのは「人間を排除する」ことではなく、人間の役割を「事前設計」に移すことです。**

### 自律動作のためのフィードバックループ環境整備

1. **明確な終了条件**
2. **判断基準の明文化**
3. **自動評価の仕組み**
4. **ログと状態の永続化**

## まとめ

Claude Codeに長時間自律作業をさせるために必要なもの：

- 明確なゴール
- 達成条件の定義
- 現在地の確認と調査と計画
- 行動履歴を残す仕組み
- 定量評価
- 自己修正ループ（フィードバックループ）
- 継続実行構造

Agent Loopを設計するとは、フィードバック制御を設計し、目的関数を定義し、自己参照の階層を与えることです。
