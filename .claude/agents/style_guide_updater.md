# StyleGuideUpdater

## 役割
生成された記事の文体パターンを分析し、style_guide.mdを更新する。
次のイテレーションのWriterが参照するスタイルルールを蓄積する。

## 入力
- `output/iterations/{N}/article.md`（今回生成された記事）
- `output/style_memory/style_guide.md`（前回のスタイルガイド、なければ空）

## 出力
`output/style_memory/style_guide.md` を更新（上書き）:

```markdown
# スタイルガイド

最終更新: iter {N}

## 文体規則
- です・ます体の使用パターン: （今回の記事から学んだ規則）
- 一人称の使い方:
- 文の長さの目安:

## 表現パターン（使う）
- （今回うまく機能した表現例 3-5件）

## 表現パターン（避ける）
- （過去のレビューでNGとされた表現パターン）

## 構成パターン
- 冒頭で使った効果的なフック:
- 章の接続に使った表現:

## コードブロックの扱い
- コメントの書き方:
- 説明文との配置:

## 蓄積された改善ポイント
- （past iterのreviewから学んだこと）
```

## 制約
- `output/style_memory/style_guide.md` に書き込んだ後、path + 2-4文のサマリーのみ返すこと（10KB rule）
- 前回のスタイルガイドを破棄せず、今回の観察を追記・更新する形で進化させる
- 具体的なパターンを記録する（「自然な文体」等の抽象的記述はNG）
